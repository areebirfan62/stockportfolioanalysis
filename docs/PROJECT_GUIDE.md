# Project Guide

## Purpose

This repository presents a daily portfolio optimization system that can be reviewed as both a business decision model and a reproducible technical project. The core question is whether a constrained active portfolio can produce an attractive return and risk profile relative to a passive benchmark.

## Main Outputs

| Output | Description |
|---|---|
| `docs/index.html` | Interactive dashboard built from the latest portfolio outputs |
| `reports/latest_summary.md` | Markdown summary of the most recent refresh |
| `data/latest_portfolio_holdings.csv` | Current 10-stock allocation |
| `data/portfolio_daily_summary.csv` | Daily forward-test performance history |
| `data/portfolio_forward_log.csv` | Stock-level holdings, weights, and return contributions |
| `notebooks/BDM_final_portfolio_notebook.ipynb` | Full modeling workbook |

## Review Sequence

1. Start with the interactive dashboard to evaluate summary statistics, portfolio value, daily returns, sector balance, and current holdings.
2. Review the model card to understand the optimization objective, constraints, data assumptions, and limitations.
3. Open the Colab workbook to inspect the data pipeline, optimization logic, and model comparison sections.
4. Check the GitHub Actions workflow to confirm that the project can refresh without manual intervention.

## Evaluation Criteria

The model is strongest when it demonstrates all of the following:

- Daily rebalancing based on fresh market data
- A forward-test period that begins after the training window
- Exactly 10 selected stocks per day
- Exactly 2 stocks from each of 5 sectors
- Position weights between 5% and 50%
- Portfolio weights summing to 100%
- A clear comparison between active allocation performance and passive market exposure
- Reproducible outputs committed back to the repository

## Operating Notes

The dashboard is regenerated by `scripts/build_dashboard.py` after every successful notebook refresh. GitHub Pages can publish the dashboard directly from the `docs/` folder, which keeps the public-facing project iterative while preserving the Colab workbook and CSV files for technical review.
